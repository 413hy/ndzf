# 账号状态、Session 和搭话配置说明

## 1. Session 文件在哪里

当前账号 Session 保存在：

`D:\test\fdzn_codex\4.1_emergency_workcopy\sessions`

每个账号通常有两类文件：

- `account_01.session`：真正的 Telegram 登录会话文件。
- `account_01.json`：这个账号对应的 API、设备信息等辅助配置。

根目录还有两个辅助文件：

- `session_config.json`：账号名到 API、设备信息的映射。
- `session_string_cache.json`：从原始 string session 转出来时保留的缓存，里面是敏感登录材料，不要发给外人。

## 2. 为什么退出后账号状态会丢

这个打包程序原本没有把“检测状态、SpamBot 结果、用户名、手机号”等账号状态完整写回数据库或配置文件，主要放在运行时缓存里。

我已在 `ui/app.js` 加了一个本机缓存补丁：程序显示账号状态后，会把最近一次状态保存到本机 `localStorage`。下次启动时，如果后端只返回“未检测/未知/空状态”，界面会先显示上一次保存的结果。

注意：这不是 Telegram 的实时检测结果，只是“上一次检测结果快照”。需要最新状态时，仍然要点程序里的账号检测功能。

## 3. 搭话功能在哪里配置

现有包里已经有“监听私信/监听群消息后自动回复”的配置入口，不需要新建后端。

进入程序后按这个顺序配置：

1. 打开左侧“监听私信/监听”页面。
2. 选择监听账号。
3. 如果要收到私信后自动回复，打开“收到私信自动回复”配置，添加回复内容，设置冷却时间和延迟。
4. 如果要命中群内关键词后在群里搭话，打开“群内回复配置”，添加回复文案或贴纸包。
5. 设置关键词、黑名单和过滤规则。
6. 保存配置，然后启动监听。

这些配置会保存到：

`D:\test\fdzn_codex\4.1_emergency_workcopy\config.json`

相关配置项包括：

- `monitor_private_reply_texts`：收到私信后的自动回复内容。
- `monitor_reply_direct_texts`：群内命中关键词后的回复文案。
- `monitor_reply_sticker_packs`：群内回复使用的贴纸包。
- `dm_direct_texts`：批量私信直发文案，也会被监听私信模块复用。

## 4. 日常使用建议

- 平时用 `启动软件.bat` 启动，避免授权校验再次去访问已经不存在的旧服务器。
- 账号检测完后再退出一次程序，确保界面缓存已写入。
- 如果要复制给同事使用，至少复制整个程序目录，不要只复制 exe。
- 不要删除 `sessions`、`session_config.json`、`session_string_cache.json`、`config.json`、`license.dat`。
